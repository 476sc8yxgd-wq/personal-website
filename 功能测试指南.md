# 个人网站功能测试指南

## 部署后测试步骤

### 1. 基础连接测试

#### 1.1 健康检查
```bash
curl http://localhost:3000/health
```
**预期结果**: 返回`OK`状态码200

#### 1.2 API健康检查
```bash
curl http://localhost:3000/api/health
```
**预期结果**: 返回JSON格式的服务器状态信息

### 2. 前端页面测试

#### 2.1 主页测试
- 访问: `http://localhost:3000/`
- **验证点**:
  - 页面正常加载
  - 导航栏显示正确
  - 页面内容完整显示

#### 2.2 关于页面测试
- 访问: `http://localhost:3000/about`
- **验证点**:
  - 页面正常加载
  - 个人信息显示正确

#### 2.3 博客页面测试
- 访问: `http://localhost:3000/blog`
- **验证点**:
  - 博客列表正常显示
  - 点击博客详情能正确跳转
- 博客详情测试: `http://localhost:3000/blog/1`
  - **验证点**: 博客详情内容正常显示

#### 2.4 问答页面测试
- 访问: `http://localhost:3000/qa`
- **验证点**:
  - 问答列表正常显示
  - 问答内容完整

### 3. 后端API测试

#### 3.1 个人资料API测试
```bash
# 获取个人资料
curl http://localhost:3000/api/profile
```
**预期结果**: 返回个人资料信息

#### 3.2 博客API测试
```bash
# 获取博客列表
curl http://localhost:3000/api/blogs

# 获取特定博客
curl http://localhost:3000/api/blogs/1
```
**预期结果**: 返回相应的博客数据

#### 3.3 问答API测试
```bash
# 获取问答列表
curl http://localhost:3000/api/qa
```
**预期结果**: 返回问答数据

### 4. 数据库连接测试

#### 4.1 检查MySQL容器状态
```bash
docker-compose ps
```
**验证点**: MySQL容器显示为"Up"状态

#### 4.2 连接数据库
```bash
docker-compose exec mysql mysql -u root -p
```
**验证点**: 能够成功连接到数据库

### 5. 容器日志检查

#### 5.1 查看应用日志
```bash
docker-compose logs web
```
**验证点**: 无严重错误信息，服务启动正常

#### 5.2 查看数据库日志
```bash
docker-compose logs mysql
```
**验证点**: 数据库初始化成功，无连接错误

### 6. 性能基本测试

#### 6.1 页面加载速度
- 打开浏览器开发者工具
- 访问各个页面，检查加载时间
- **预期结果**: 页面加载时间在合理范围内

#### 6.2 API响应时间
- 使用浏览器开发者工具的Network标签
- 观察API请求响应时间
- **预期结果**: API响应时间在可接受范围内

## 常见问题排查

### 1. 页面无法访问
- 检查容器是否运行: `docker-compose ps`
- 检查端口是否被占用: `netstat -tulpn | grep :3000`
- 查看容器日志: `docker-compose logs web`

### 2. API请求失败
- 检查后端服务日志: `docker-compose logs web`
- 验证API端点路径是否正确
- 检查数据库连接是否正常

### 3. 数据库连接失败
- 检查MySQL容器状态: `docker-compose ps mysql`
- 验证数据库配置信息
- 检查数据库初始化日志

当所有测试项目都通过后，可以确认部署成功，网站已准备好投入使用。
